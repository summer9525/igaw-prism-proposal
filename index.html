프로젝트 로드맵: 2단계 구조

PRISM은 크게 두 단계로 나뉩니다. 1단계에서 오디언스 태깅으로 자사 데이터를 구축하고, 2단계에서 그 데이터를 활용해 경쟁사 분석 리포트를 생성합니다.

01
PHASE 1

오디언스 태깅을 통한 자사 데이터 구축

기존 Raw Data를 R-Tag, F-Tag로 정제하고, 업종+카테고리별 주요 오디언스(S-Tag)를 사전 구축합니다.

R-Tag Raw Data 태깅 (w. Kiro)
F-Tag 고객 속성 페르소나 생성
S-Tag 업종별 주요 오디언스 × 전략 사전 구축
02
PHASE 2

자사 데이터를 활용한 경쟁사 분석 리포트

업종+카테고리 선택 → S-Tag(오디언스×전략) 선택 → 기본 분석 + 경쟁사 분석 리포트를 자동 생성합니다.

① 업종 + 카테고리 선택
② S-Tag 선택 (오디언스 × 전략)
③ 기본 분석 + 경쟁사 분석 리포트 제공

💡 핵심 포인트: 기존에는 '분석'에서 끝나면 현상만 알려주는 느낌이었지만, PRISM은 '경쟁 분석'이 메인이고, 그것을 뒷받침하는 것이 오디언스입니다. 오디언스의 신뢰도를 경쟁 분석 결과로 증명하는 구조입니다.

1

기존 컨설팅 업체의 한계점

기존 컨설팅(맥킨지, BCG 등)이나 시장조사 업체를 이용할 때 기업들이 느끼는 고질적인 문제입니다. 이를 해결하는 것이 PRISM의 '파괴적 혁신' 포인트입니다.

⏳
휘발성 및 적시성 부족

보고서가 나오는 시점에는 이미 과거 데이터. 실시간으로 변하는 모바일/커머스 시장 대응에 한계가 있습니다.

📋
표본의 한계

설문조사 기반 리포트는 실제 행동 데이터가 아닌 '기억'에 의존하므로 정확도가 떨어집니다.

💸
높은 비용 대비 낮은 실행력

수억 원의 비용을 들여 전략을 짜도, "내일 아침에 누구에게 어떤 메시지를 보낼지"에 대한 실행 수단은 제공하지 못합니다.

F-Tag Feature: 고객 속성 페르소나

R-Tag를 조합하여 고객의 '페르소나와 라이프스타일'을 정의하는 속성 레이어. 컨설팅의 '가설'을 검증하는 핵심 지표입니다.

초정밀 직장인 프로파일링

'여의도 금융권 고연봉자', '판교 대기업 IT 기획자', '울산 제조 현장직'

GPS + 직종 + 회사 규모 + 연봉 결합

쇼핑 페르소나

'계절가전 얼리어답터', '마트 주중 방문형 체리피커', '프리미엄 카테고리 헤비 쇼퍼'

커머스 카테고리 + 평균 단가 결합

경쟁사 관계 정의

'경쟁사 A 전용 유저', '우리 앱-경쟁사 B 교차 사용자', '최근 경쟁사로의 이탈 징후군'

앱 사용성 + 이탈 경로 결합

관심사 세그먼트

대분류 25개 및 중분류 214개 카테고리별 고관여 유저

앱 사용 패턴 + 커머스 데이터 결합

S-Tag Strategy: 오디언스 × 전략 목적

S-Tag는 "누구를(오디언스)" + "어떻게 할 것인가(전략 목적)"의 조합입니다. 업종+카테고리별로 사전 구축된 오디언스에 전략 방향을 결합하여, 선택 즉시 경쟁사 분석 리포트가 생성됩니다.

오디언스만으로는 '현상 파악'에 그치고, 전략만으로는 '누구에게'가 빠집니다. S-Tag는 이 둘을 하나로 묶어 실행 가능한 단위로 만듭니다.

PART 7 VIP 이탈 방지 — Win-back 시나리오

이 오디언스 내 상위 5% VIP 고객 중 경쟁사 이탈 징후가 감지된 유저에 대한 선제 방어 전략입니다.

🚨 이탈 징후 감지 기준
  • 최근 30일 우리 앱 사용 시간 40% 이상 감소
  • 동기간 밀크T 또는 엘리하이 사용 시간 30% 이상 증가
  • 우리 앱 결제 빈도 주 2회 → 주 0.5회 이하로 감소
  • 경쟁사 앱 신규 설치 이력 (최근 14일 내)
🛡️ 4단계 Win-back 시나리오
  • D+1: 개인화 푸시 — "자녀의 학습 리포트가 업데이트되었습니다"

  • D+14: 오프라인 연계 — 대치동 학습 상담 예약 + 교재 증정

IGAWorks 제품 시너지 맵

PRISM은 단독 제품이 아닌, 기존 IGAWorks 제품군과 결합하여 '분석 → 실행'의 풀 퍼널을 완성합니다.

📊
모바일인덱스 + PRISM
앱 분석 + 경쟁사 분석

모바일인덱스의 앱 시장 데이터에 PRISM의 ADID 기반 경쟁사 오디언스 분석을 결합. 시장 트렌드 파악과 동시에 경쟁사 고객 세그먼트까지 심층 분석이 가능합니다.

  • 앱 MAU/DAU 트렌드 + 경쟁사 오디언스 프로파일
  • 카테고리별 점유율 + 이탈 경로 추적
  • 시장 리포트 + 전략 리포트 원스톱 제공
📡
트웍 / MI Ads + PRISM
경쟁사 분석 + 경쟁사 오디언스 광고

PRISM이 발견한 경쟁사 오디언스를 트웍/MI Ads를 통해 즉시 광고 타겟으로 전환. 분석에서 실행까지 끊김 없는 파이프라인을 구축합니다.

  • 경쟁사 헤비유저 ADID → DSP 타겟 광고 즉시 집행
  • 이탈 징후 고객 → 리타겟팅 캠페인 자동 연결
  • 리포트 내 "이 오디언스에 광고하기" 원클릭 실행
🔬
디파이너리 + PRISM
자사 데이터 + 경쟁사 데이터 통합 분석

디파이너리의 자사 1st Party 데이터에 PRISM의 경쟁사 3rd Party 데이터를 결합. 자사 고객이 경쟁사에서 어떻게 행동하는지까지 볼 수 있습니다.

  • 자사 CRM 데이터 + 경쟁사 앱 사용 패턴 결합
  • 자사 VIP의 경쟁사 이탈 징후 실시간 감지
  • 자사/경쟁사 통합 고객 여정 분석

✅ 시너지 핵심: PRISM 단독으로는 '분석'이지만, 모바일인덱스와 붙으면 '시장+경쟁사', 트웍/MI Ads와 붙으면 '분석+실행', 디파이너리와 붙으면 '자사+경쟁사 통합'. 고객사 니즈에 따라 조합을 달리하여 패키지 판매가 가능합니다.

5

Kiro 활용 가이드: 데이터팀 × 마케팅팀

Kiro는 단순한 코딩 도구가 아니라, '전략적 명세(Spec)를 이해하고 검증하는 에이전트'입니다. 데이터팀과 마케팅팀이 각각 다른 방식으로 활용합니다.

데이터팀 R-F-S 태깅 시스템의 무결성 확보
① R→F 변환 로직 설계 (Stamper 단계)
GPS + 회사 명세 → 직장인 페르소나 생성

"GPS 좌표와 회사 명세를 매칭해 '직장인 페르소나'를 생성하라"는 스펙을 Kiro에게 전달. 데이터 정제 및 태깅 알고리즘 초안을 생성하고, 데이터 간 의존성을 분석합니다.

→ 복잡한 SQL/Python 로직 작성 시간 단축 + 태깅 누락 사전 방지

② 데이터 신뢰도 검증 (Quality Assurance)
엣지 케이스 시뮬레이션

CEO 책상에 오를 데이터인 만큼, 수천 개의 엣지 케이스 테스트를 Kiro에게 맡깁니다. 예: "주말에 회사 근처를 지나는 관광객을 '출퇴근 직장인'으로 오분류하지 않는가?"

→ 사람이 놓치기 쉬운 데이터 오차를 잡아내어 PRISM의 대외 신뢰도 확보

마케팅팀 리포트 자동화 및 전략 시나리오 고도화
① S-Tag 기반 리포트 생성 엔진
Executive Summary 자동 생성

분석 결과를 기반으로 CEO가 즉시 읽을 수 있는 Executive Summary와 전략 제안 텍스트를 자동 생성하는 엔진을 설계합니다. 조건에 따라 리포트의 텍스트와 표가 유동적으로 변합니다.

→ 컨설팅 보고서 수작업 시간을 제로(0)에 가깝게 단축

② 타겟팅 시나리오 시뮬레이션 (Strategy Finder)
최적 오디언스 조합 탐색

"경쟁사 A의 VIP 유저를 우리 앱으로 데려오기 위한 최적의 오디언스 조합"을 시뮬레이션. 특정 F-Tag 조합 적용 시 예상 타겟 모수와 마케팅 반응률을 가상 테스트합니다.

→ 광고주에게 넘기기 전 전략의 예상 ROI를 미리 검증하여 설득력 강화
